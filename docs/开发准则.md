# 开发准则

## 1. 环境

- Go latest
- Flutter latest
- 依赖库保持最新稳定版

---

## 2. 代码规范

### 2.1 注释

所有注释使用中文。中英混杂时，中英文之间加空格。

### 2.2 Go

```go
// 接口定义依赖
type ChatService interface {
    SendMessage(ctx context.Context, msg *Message) error
}

// 错误定义
var ErrNotFound = errors.New("resource not found")

// context 传递请求上下文
func (s *service) SendMessage(ctx context.Context, msg *Message) error {}
```

### 2.3 Flutter/Dart

Clean Architecture 分层：`data` → `domain` → `presentation`

```dart
// Entity
class User {
  final String id;
  final String username;
  const User({required this.id, required this.username});
}

// Repository Interface
abstract class AuthRepository {
  Future<Either<Failure, User>> login(String email, String password);
}

// Use Case
class LoginUseCase {
  final AuthRepository repository;
  Future<Either<Failure, User>> call(LoginParams params) =>
      repository.login(params.email, params.password);
}
```

## 3. 目录结构

### 3.1 环境变量


## 5. gRPC 规范

### 错误处理

```go
// 参数错误
return nil, status.Error(codes.InvalidArgument, "user_id 不能为空")

// 权限错误
return nil, status.Error(codes.PermissionDenied, "无权限")
```

### 常用状态码

| 状态码 | 用途 |
|--------|------|
| OK | 成功 |
| InvalidArgument | 参数错误 |
| Unauthenticated | 未认证 |
| PermissionDenied | 无权限 |
| NotFound | 不存在 |
| Internal | 服务器错误 |

## 6. 日志规范

基于 Go `log/slog`，使用结构化 JSON 日志。

### 日志级别

| 级别 | 用途 |
|------|------|
| DEBUG | 调试信息（生产关闭） |
| INFO | 正常业务流程 |
| WARN | 潜在问题 |
| ERROR | 错误 |

### 使用示例

```go
// 简单日志
log.Info("服务启动", slog.Int("port", 50052))

// 带上下文
log.WithContext(ctx).Info("消息发送成功",
    slog.String("conversation_id", convID),
)

// 错误日志
log.WithContext(ctx).Error("发送失败",
    slog.Any("error", err),
)
```

### 最佳实践

```go
// ✅ 结构化字段
log.Info("用户登录", slog.String("user_id", userID))

// ❌ 拼接字符串
log.Info(fmt.Sprintf("用户 %s 登录", userID))

// ❌ 记录敏感信息
log.Info("登录", slog.String("password", password))
```

## 7. 调试

### grpcurl 安装

```bash
brew install grpcurl
```

### 常用命令

```bash
# 注册
grpcurl -plaintext \
  -d '{"username":"test","email":"test@example.com","password":"123456"}' \
  localhost:50053 auth.AuthService/Register

# 登录
grpcurl -plaintext \
  -d '{"email":"test@example.com","password":"123456"}' \
  localhost:50053 auth.AuthService/Login

# 带 Token 请求
grpcurl -plaintext \
  -H "authorization: Bearer $TOKEN" \
  -d '{"user_id":"xxx"}' \
  localhost:50053 feed.FeedService/GetFollowingFeed
```

### 直连服务（绕过 Gateway）

```bash
# Auth 服务
grpcurl -plaintext localhost:50054 auth.AuthService/Login

# Chat 服务
grpcurl -plaintext localhost:50052 chat.ChatService/GetConversations
```

### 查看日志

```bash
# 所有服务
docker compose -f infra/docker-compose.yml logs -f

# 单个服务
docker compose -f infra/docker-compose.yml logs -f chat

# Web 界面
open http://localhost:9999
```

### 数据库检查

```bash
docker exec -it postgres psql -U lesser -d lesser_db
SELECT id, username FROM users;
```

## 8. CLI 工具

所有脚本功能已完全内置到 CLI 中，无需依赖外部 shell 脚本。

```bash
# 安装
cargo install --path infra/cli

# 服务管理
devlesser start              # 启动所有
devlesser start infra        # 只启动基础设施
devlesser start service      # 只启动后端
devlesser stop
devlesser status

# 初始化（含 hosts 配置）
devlesser init               # 完整初始化
devlesser init --skip-hosts  # 跳过 hosts 配置

# Proto 生成
devlesser proto              # 全部 (别名: devlesser gen)
devlesser proto go           # 仅 Go
devlesser proto dart         # 仅 Dart

# 测试
devlesser test               # 运行所有测试
devlesser test services      # 仅服务测试
devlesser test search        # 仅搜索测试

# hosts 配置
devlesser hosts              # 配置本地域名 (需要 sudo)

# 生产环境
devlesser prod start         # 启动生产环境
devlesser prod stop          # 停止生产环境
devlesser prod deploy        # 部署更新
devlesser prod backup        # 备份数据库
devlesser prod logs          # 查看日志

# 清理
devlesser clean
devlesser clean volumes
```

## 9. 安全规范

### 认证

- JWT 认证
- Access Token：开发 1h，生产 30min
- Refresh Token：开发 7d，生产 1d

### 数据安全

- 密码使用 bcrypt/argon2
- 敏感数据传输使用 HTTPS
- 日志不记录敏感信息

### 输入验证

- 所有用户输入必须验证
- 参数化查询防 SQL 注入
- HTML 转义防 XSS
