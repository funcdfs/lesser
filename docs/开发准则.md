## 2. 代码规范

### 2.1 注释

所有注释使用中文。中英混杂时，中英文之间加空格。数字和汉字之间也要加空格。

### 2.2 Go

```go
// 接口定义依赖
type ChatService interface {
    SendMessage(ctx context.Context, msg *Message) error
}

// 错误定义
var ErrNotFound = errors.New("resource not found")

// context 传递请求上下文
func (s *service) SendMessage(ctx context.Context, msg *Message) error {}
```

### 2.3 Flutter/Dart

分层架构：`pages` → `handler` → `data_access` → `models`

```dart
// Model
class User {
  final String id;
  final String username;
  const User({required this.id, required this.username});
}

// Handler (业务逻辑)
class AuthHandler extends ChangeNotifier {
  final AuthDataAccess _dataAccess;
  Future<void> login(String email, String password) async {}
}

// DataAccess (gRPC 调用)
class AuthDataAccess {
  final GrpcChannel _channel;
  Future<LoginResponse> login(LoginRequest request) async {}
}
```

### 2.4 Flutter UI 开发风格

#### 核心原则

- **脱离 Material 限制**：不依赖 Material 组件的默认样式，使用纯自定义实现
- **最小化 ThemeData**：只保留必要配置，禁用多余效果
- **私有化专用组件**：只在单一页面使用的组件定义为私有类，不抽取到公共目录

#### 交互动效规范

```dart
// 缩放动画：按下缩小，松开弹回
_scale = Tween(begin: 1.0, end: 0.85).animate(
  CurvedAnimation(parent: _ctrl, curve: Curves.easeOutCubic),
);

// 触觉反馈
HapticFeedback.lightImpact();

// 选中状态光晕（典雅、克制）
BoxShadow(
  color: color.withValues(alpha: 0.12),
  blurRadius: 8,
  spreadRadius: 1,
)

// 选中状态：线条加粗，不填充
final strokeWidth = isSelected ? 2.4 : 1.6;
```

#### 自定义图标

使用 SVG Path 字符串 + CustomPainter，避免引入图标库：

```dart
// 图标定义（24x24 viewBox）
const _iconFeed = 'M4 4H16 M4 9H20 M4 14H20 M4 19H14';

// 绘制器
class _IconPainter extends CustomPainter {
  final String path;
  final Color color;
  final double strokeWidth;
  
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()
      ..color = color
      ..style = PaintingStyle.stroke
      ..strokeWidth = strokeWidth
      ..strokeCap = StrokeCap.round
      ..strokeJoin = StrokeJoin.round;
    canvas.drawPath(_parse(path, size), paint);
  }
}
```

#### 禁用 Material 默认效果

```dart
ThemeData(
  useMaterial3: true,
  splashFactory: NoSplash.splashFactory,
  highlightColor: Colors.transparent,
  splashColor: Colors.transparent,
)
```

#### 文件组织

```
// ✅ 专用组件放在使用它的文件内
lib/features/home/pages/home_page.dart
├── HomePage (公开)
├── _BottomNavBar (私有)
├── _NavItem (私有)
└── _IconPainter (私有)

// ❌ 不要为只用一次的组件创建独立文件
lib/pkg/ui/widgets/bottom_nav_bar.dart  // 错误
```

## 3. 目录结构

### 3.1 Go 后端

```
service/<name>/
├── cmd/                # 入口
│   └── main.go
├── internal/
│   ├── handler/        # gRPC 处理器（协议对接、参数转换）
│   ├── logic/          # 核心业务规则（权限判断、缓存策略）
│   ├── remote/         # 外部服务调用（跨服务 gRPC 调用）
│   ├── data_access/    # 数据库存取（SQL/NoSQL 操作）
│   └── messaging/      # 异步消息发布/订阅（RabbitMQ）
```

## 5. 新增路由流程

```
1. Proto 定义
   └── protos/<service>/<service>.proto
       ├── 添加 message 定义
       └── 添加 rpc 方法

2. 后端服务
   ├── Gateway 路由配置
   │   └── service/gateway/internal/router/
   ├── Service 业务处理
   │   ├── service/<service>/internal/handler/
   │   ├── service/<service>/internal/logic/
   │   └── service/<service>/internal/data_access/
   └── 跨服务调用（如需要）
       └── service/<service>/internal/remote/

3. 网关配置（如需新路径前缀）
   └── infra/gateway/dynamic/routes.yml

4. 客户端
   └── Flutter
       ├── lib/features/<module>/data_access/
       ├── lib/features/<module>/models/
       ├── lib/features/<module>/handler/
       └── lib/features/<module>/pages/
```

## 7. gRPC 规范

### 错误处理

```go
// 参数错误
return nil, status.Error(codes.InvalidArgument, "user_id 不能为空")

// 权限错误
return nil, status.Error(codes.PermissionDenied, "无权限")
```

### 常用状态码

| 状态码 | 用途 |
|--------|------|
| OK | 成功 |
| InvalidArgument | 参数错误 |
| Unauthenticated | 未认证 |
| PermissionDenied | 无权限 |
| NotFound | 不存在 |
| Internal | 服务器错误 |

## 8. 日志规范

基于 Go `log/slog`，使用结构化 JSON 日志。

### 日志级别

| 级别 | 用途 |
|------|------|
| DEBUG | 调试信息（生产关闭） |
| INFO | 正常业务流程 |
| WARN | 潜在问题 |
| ERROR | 错误 |

### 使用示例

```go
// 简单日志
log.Info("服务启动", slog.Int("port", 50052))

// 带上下文
log.WithContext(ctx).Info("消息发送成功",
    slog.String("conversation_id", convID),
)

// 错误日志
log.WithContext(ctx).Error("发送失败",
    slog.Any("error", err),
)
```

### 最佳实践

```go
// ✅ 结构化字段
log.Info("用户登录", slog.String("user_id", userID))

// ❌ 拼接字符串
log.Info(fmt.Sprintf("用户 %s 登录", userID))

// ❌ 记录敏感信息
log.Info("登录", slog.String("password", password))
```

## 11. 安全规范

### 认证

- JWT 认证
- Access Token：开发 1h，生产 30min
- Refresh Token：开发 7d，生产 1d

### 数据安全

- 密码使用 bcrypt/argon2
- 敏感数据传输使用 HTTPS
- 日志不记录敏感信息

### 输入验证

- 所有用户输入必须验证
- 参数化查询防 SQL 注入
- HTML 转义防 XSS
