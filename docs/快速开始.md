## 系统架构

```
                        互联网
                          ↓
                    APISIX 网关 (9080)
                     ↙         ↖
              Django (8000)    etcd (配置)
                ↙
          PostgreSQL (5432)
               + Redis (6379)
```

### 网络通讯

- **客户端 → APISIX (9080)**：所有 API 请求先经 APISIX 网关
- **APISIX → etcd (2379)**：从 etcd 读取动态路由配置
- **APISIX → Django (8000)**：根据路由规则转发到 Django
- **Django ↔ PostgreSQL (5432)**：存储业务数据
- **Django ↔ Redis (6379)**：存储会话、缓存等

## 快速开始（5 分钟）

### 前置条件

- Docker & Docker Compose 已安装
- Git 已安装
- 足够的磁盘空间（约 2GB）

### 启动开发环境

#### 方案 A：完整环境（包括 APISIX 网关）【推荐】

```bash
# 进入项目目录
cd /path/to/lesser

# 启动所有服务
docker-compose -f infra/docker-compose-apisix.yml up -d

# 初始化数据库（创建表、超级用户等）
docker-compose -f infra/docker-compose-apisix.yml exec django python manage.py migrate
docker-compose -f infra/docker-compose-apisix.yml exec django python manage.py createsuperuser

# 检查服务状态
docker-compose -f infra/docker-compose-apisix.yml ps

# 查看日志
docker-compose -f infra/docker-compose-apisix.yml logs -f django
```

#### 方案 B：简化环境（仅数据库 + Django）

```bash
# 启动基础服务
docker-compose -f infra/docker-compose.yml up -d

# 初始化数据库
docker-compose -f infra/docker-compose.yml exec django python manage.py migrate
docker-compose -f infra/docker-compose.yml exec django python manage.py createsuperuser
```

## 开发工作流

### 场景 1：修改 Django 代码

**步骤**：
1. 修改 `backend/django_code/` 下的代码
2. Django 服务会自动重启（`--reload` 标志）
3. 刷新浏览器看到更新

**示例**：添加新的 API 端点
```python
# backend/django_code/app/users/views.py
from rest_framework.response import Response
from rest_framework.decorators import api_view

@api_view(['GET'])
def hello(request):
    return Response({"message": "Hello World"})

# 在 backend/django_code/app/users/urls.py 中添加
path('hello/', hello, name='hello')
```

访问：`curl http://localhost:9080/api/users/hello/`

### 场景 2：修改数据库模型

**步骤**：
1. 修改模型文件 (`models/user.py` 等)
2. 创建迁移：`docker-compose exec django python manage.py makemigrations`
3. 应用迁移：`docker-compose exec django python manage.py migrate`
4. 重启 Django（自动）

**示例**：为 User 模型添加字段
```python
# backend/django_code/app/users/models/user.py
class User(models.Model):
    # ... 现有字段 ...
    phone = models.CharField(max_length=20, blank=True)  # 新增字段

# 终端运行
docker-compose -f infra/docker-compose-apisix.yml exec django python manage.py makemigrations
docker-compose -f infra/docker-compose-apisix.yml exec django python manage.py migrate
```

### 场景 3：修改 Flutter 代码

**步骤**：
1. 修改 `frontend/lib/` 下的代码
2. 保存文件，IDE 会自动热重载
3. 或使用快捷键：
   - `r` 热重载（保留应用状态）
   - `R` 热重启（清除应用状态）

## 新增功能完整流程

### 需求：添加一个"获取用户列表"的 API

#### 1️⃣ 后端实现（Django）

**定义数据模型** (`backend/django_code/app/users/models/user.py`)：
```python
from django.db import models

class User(models.Model):
    username = models.CharField(max_length=100, unique=True)
    email = models.EmailField()
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        db_table = 'users_user'
```

**定义序列化器** (`backend/django_code/app/users/serializers/user.py`)：
```python
from rest_framework import serializers
from ..models import User

class UserSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ['id', 'username', 'email', 'created_at']
```

**实现视图** (`backend/django_code/app/users/views/user.py`)：
```python
from rest_framework.viewsets import ModelViewSet
from ..models import User
from ..serializers import UserSerializer

class UserViewSet(ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSerializer
```

**注册路由** (`backend/django_code/app/users/urls.py`)：
```python
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import UserViewSet

router = DefaultRouter()
router.register('', UserViewSet, basename='user')

urlpatterns = [
    path('', include(router.urls)),
]
```

**创建迁移**：
```bash
docker-compose -f infra/docker-compose-apisix.yml exec django python manage.py makemigrations
docker-compose -f infra/docker-compose-apisix.yml exec django python manage.py migrate
```

#### 2️⃣ 前端实现（Flutter）

**定义业务模型** (`frontend/lib/domain/entities/user.dart`)：
```dart
class User {
  final int id;
  final String username;
  final String email;
  
  User({
    required this.id,
    required this.username,
    required this.email,
  });
}
```

**定义 DTO** (`frontend/lib/data/models/user_model.dart`)：
```dart
class UserModel {
  final int id;
  final String username;
  final String email;
  
  factory UserModel.fromJson(Map<String, dynamic> json) {
    return UserModel(
      id: json['id'],
      username: json['username'],
      email: json['email'],
    );
  }
  
  User toEntity() => User(
    id: id,
    username: username,
    email: email,
  );
}
```

**实现 Repository** (`frontend/lib/data/repositories/user_repository.dart`)：
```dart
class UserRepository {
  Future<List<User>> getUsers() async {
    final response = await http.get('/api/users/');
    return (response.data as List)
        .map((u) => UserModel.fromJson(u).toEntity())
        .toList();
  }
}
```

**创建状态管理** (`frontend/lib/feature/user/presentation/providers/user_provider.dart`)：
```dart
final userListProvider = FutureProvider<List<User>>((ref) {
  final repo = ref.watch(userRepositoryProvider);
  return repo.getUsers();
});
```

**实现 UI** (`frontend/lib/feature/user/presentation/pages/user_list_page.dart`)：
```dart
class UserListPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final users = ref.watch(userListProvider);
    
    return users.when(
      data: (users) => ListView(
        children: users.map((u) => ListTile(
          title: Text(u.username),
          subtitle: Text(u.email),
        )).toList(),
      ),
      loading: () => CircularProgressIndicator(),
      error: (err, st) => Text('Error: $err'),
    );
  }
}
```

## 问题排查

### Q1：服务无法启动，报数据库连接错误

**症状**：
```
django | Error connecting to PostgreSQL
postgres | connection refused
```

**解决**：
```bash
# 检查 PostgreSQL 容器是否运行
docker ps | grep postgres

# 检查 PostgreSQL 日志
docker logs postgres

# 清理并重新启动
docker-compose -f infra/docker-compose-apisix.yml down -v
docker-compose -f infra/docker-compose-apisix.yml up -d
```

### Q2：API 请求返回 404

**症状**：`curl http://localhost:9080/api/users/` 返回 404

**解决**：
1. 检查 APISIX 路由配置：
   ```bash
   # 访问 APISIX Dashboard 查看路由
   open http://localhost:9000/
   ```
2. 检查 Django 服务是否运行：
   ```bash
   docker-compose -f infra/docker-compose-apisix.yml logs django
   ```
3. 直接访问 Django：
   ```bash
   curl http://localhost:8001/api/users/
   ```

### Q3：迁移失败，提示字段冲突

**症状**：
```
Error: Conflicting models: auth.user vs users.customuser
```

**解决**：
```bash
# 删除迁移文件（除 __init__.py），重新生成
rm backend/django_code/app/users/migrations/000*.py
docker-compose exec django python manage.py makemigrations
docker-compose exec django python manage.py migrate
```

### Q4：热重载不工作

**症状**：修改 Django 代码后需要手动重启

**解决**：
1. 确认命令包含 `--reload`：
   ```bash
   docker-compose logs django | grep reload
   ```
2. 如果没有，重启容器：
   ```bash
   docker-compose restart django
   ```

## 网关配置（APISIX）

### 查看当前路由

```bash
# 通过 Dashboard (推荐)
open http://localhost:9000/

# 通过 API
curl http://localhost:9180/apisix/admin/routes
```

### 添加新路由

使用 Dashboard：
1. 访问 http://localhost:9000/
2. 点击 "路由"
3. 点击 "创建路由"
4. 填写：
   - Name: `api_users`
   - Hosts: 留空
   - Path: `/api/users/*`
   - Upstream: `http://django:8000`
5. 保存

或使用 API：
```bash
curl -X POST http://localhost:9180/apisix/admin/routes \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "api_users",
    "uri": "/api/users/*",
    "upstream": {
      "nodes": {
        "django:8000": 1
      }
    }
  }'
```
