# 脚本使用完整指南

## 快速开始

### 一行启动所有环境
```bash
./dev.sh
```

### 常用命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `./dev.sh` | 启动完整后端环境 | `./dev.sh` |
| `./dev.sh logs` | 查看所有服务日志 | `./dev.sh logs` |
| `./dev.sh logs django` | 查看 Django 日志 | `./dev.sh logs django` |
| `./dev.sh restart django` | 重启 Django 服务 | `./dev.sh restart django` |
| `./dev.sh stop` | 停止所有服务 | `./dev.sh stop` |
| `./dev.sh clean` | 清理所有数据（危险！） | `./dev.sh clean` |
| `./dev.sh help` | 显示帮助 | `./dev.sh help` |

## 脚本目录结构

```
scripts/
├── dev.sh                    # 主入口脚本（统一命令）
├── dev/                      # 开发环境脚本
│   ├── start.sh             # 启动所有服务
│   ├── stop.sh              # 停止所有服务
│   ├── restart.sh           # 重启指定服务
│   ├── logs.sh              # 查看服务日志
│   └── clean.sh             # 清理所有数据
├── db/                       # 数据库管理脚本
│   ├── init.sh              # 数据库初始化
│   └── query.sh             # 数据库查询工具
└── utils/                    # 工具脚本
    ├── colors.sh            # 彩色输出工具
    └── diagnostic.sh        # 诊断和监控工具
```

## 详细说明

### dev/start.sh - 启动服务
启动完整的后端环境，包括：
- Docker 容器启动
- 数据库迁移
- 默认用户创建
- 服务健康检查

```bash
# 通过主脚本调用
./dev.sh start

# 或直接调用
./scripts/dev/start.sh
```

**输出示例：**
```

─ 检查依赖
✓ 所有依赖检查通过

─ 启动 Docker 服务
ℹ 启动 Docker 容器...
✓ Docker 服务启动成功

─ 初始化数据库
ℹ 运行数据库迁移...
ℹ 创建超级用户...
✓ 所有服务已启动！

📋 服务地址:
   Django API:       http://localhost:8000
   Django Admin:     http://localhost:8000/admin/
   PostgreSQL:       localhost:5432
   Redis:            localhost:6379
   APISIX Dashboard: http://localhost:9000
```

### dev/logs.sh - 查看日志

查看各个服务的实时日志。

```bash
# 查看所有服务日志
./dev.sh logs

# 查看 Django 日志
./dev.sh logs django

# 查看 PostgreSQL 日志
./dev.sh logs postgres

# 可用的服务: django, postgres, redis, apisix, etcd, all
```

**日志颜色说明：**
- 蓝色: 信息消息
- 绿色: 成功消息
- 黄色: 警告消息
- 红色: 错误消息

### dev/restart.sh - 重启服务

重启指定的单个服务，不影响其他服务。

```bash
# 重启 Django（代码修改后通常需要）
./dev.sh restart django

# 重启 PostgreSQL
./dev.sh restart postgres

# 重启 Redis
./dev.sh restart redis

# 重启 APISIX
./dev.sh restart apisix

# 重启 etcd
./dev.sh restart etcd
```

### dev/stop.sh - 停止服务

停止所有容器，但保留数据。

```bash
./dev.sh stop
```

使用 `./dev.sh start` 重新启动。

### dev/clean.sh - 清理数据（危险）

⚠️ **警告：这个命令会删除所有数据库内容！**

```bash
./dev.sh clean
```

会提示确认，输入 `yes` 才会执行。

清理以下内容：
- 所有 Docker 容器
- PostgreSQL 数据
- Redis 数据

### db/init.sh - 数据库初始化

手动初始化数据库（通常在 `./dev.sh start` 时自动执行）。

```bash
# 初始化数据库
source ./scripts/db/init.sh
init_database

# 重置数据库（删除所有数据）
reset_database
```

### db/query.sh - 数据库查询工具

提供快速访问 PostgreSQL 的工具函数。

```bash
source ./scripts/db/query.sh

# 连接到 PostgreSQL 交互式终端
psql_connect

# 列出所有数据库
psql_list_databases

# 列出指定数据库的表
psql_list_tables django_db

# 执行 SQL 查询
psql_query postgres "SELECT version();"

# 备份数据库
psql_backup postgres ./backup.sql
```

### utils/diagnostic.sh - 诊断工具

系统诊断和监控工具。

```bash
source ./scripts/utils/diagnostic.sh

# 检查所有服务健康状态
health_check

# 查看最新 50 行 Django 日志
tail_logs django 50

# 进入 Django 容器
shell_into_container lesser-django

# 显示容器资源使用情况
resource_usage
```

## 常见工作流

### 开发流程

```bash
# 1. 启动环境
./dev.sh

# 2. 在另一个终端查看日志
./dev.sh logs django

# 3. 编辑代码（自动热重载）

# 4. 代码有问题？检查日志
./dev.sh logs django

# 5. 手动重启服务（如果需要）
./dev.sh restart django

# 6. 完成后停止服务
./dev.sh stop
```

### 调试数据库

```bash
# 1. 连接到 PostgreSQL
source ./scripts/db/query.sh
psql_connect

# 或列出表
psql_list_tables

# 2. 查询数据
psql_query django "SELECT * FROM auth_user LIMIT 5;"

# 3. 备份数据
psql_backup django
```

### 快速重置

```bash
# 清理所有内容（谨慎！）
./dev.sh clean

# 重新启动
./dev.sh start
```

## 环境变量

脚本会自动读取 `.env` 文件中的配置（如果存在）。

常见的环境变量：
- `DOCKER_COMPOSE_FILE`: docker-compose 文件路径
- `DJANGO_CONTAINER`: Django 容器名称（默认: `lesser-django`）
- `POSTGRES_CONTAINER`: PostgreSQL 容器名称（默认: `lesser-postgres`）

## 故障排除

### "Docker 命令不存在"
```bash
# 安装 Docker Desktop 或 Docker CLI
# macOS: brew install --cask docker
# Linux: sudo apt-get install docker.io docker-compose
```

### 端口被占用
```bash
# 查看占用的端口
lsof -i :8000   # Django
lsof -i :5432   # PostgreSQL
lsof -i :6379   # Redis
lsof -i :9000   # APISIX

# 强制停止占用进程
kill -9 <PID>
```

### 容器启动失败
```bash
# 查看详细日志
docker logs lesser-django

# 检查容器状态
docker ps -a

# 查看诊断信息
source ./scripts/utils/diagnostic.sh
health_check
```

### 数据库连接失败
```bash
# 等待 PostgreSQL 就绪
docker exec lesser-postgres pg_isready -h localhost -U postgres

# 或手动等待后重新初始化
./dev.sh restart postgres
sleep 5
source ./scripts/db/init.sh
init_database
```

### 自定义脚本扩展

创建新脚本时遵循以下约定：

```bash
#!/bin/bash
set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(dirname "$(dirname "$SCRIPT_DIR")")"
UTILS_DIR="$(dirname "$SCRIPT_DIR")/../utils"

# 导入颜色工具
source "$UTILS_DIR/colors.sh"

# 定义主函数
my_function() {
    print_section "操作标题"
    print_subsection "步骤描述"
    print_info "信息"
    print_success "成功"
    print_warning "警告"
    print_error "错误"
}

export -f my_function
```
