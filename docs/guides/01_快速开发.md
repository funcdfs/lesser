# 🚀 Lesser 项目快速开发指南

## 一句话启动

```bash
./dev.sh
```

就这么简单！所有的后端服务都会自动启动。

---

## 常用命令速查表

| 需求 | 命令 | 说明 |
|------|------|------|
| **启动所有服务** | `./dev.sh` | 默认命令，启动完整环境 |
| **查看所有日志** | `./dev.sh logs` | 实时日志流（Ctrl+C 停止） |
| **查看 Django 日志** | `./dev.sh logs django` | 只看 Django 相关日志 |
| **重启 Django** | `./dev.sh restart django` | 代码改动后需要重启 |
| **停止所有服务** | `./dev.sh stop` | 优雅关闭，数据保留 |
| **清理所有数据** | `./dev.sh clean` | ⚠️ 危险！删除所有数据 |
| **显示帮助** | `./dev.sh help` | 完整命令列表 |

## 🔥 开发工作流

### 第一次开发

```bash
# 1. 启动环境
./dev.sh

# 2. 打开另一个终端，看日志
./dev.sh logs django

# 3. 开始编写代码
# Django 会自动热重载（代码更新后自动重启）

# 4. 测试 API
curl http://localhost:8000/api/users/

# 5. 完成后停止
./dev.sh stop
```

### 热重载工作原理

当你修改 Django 代码时，会自动触发：

```
修改文件 → Django 检测变化 → 自动重启 → 刷新浏览器看效果
```

**完全自动，无需手动操作！**

### 遇到问题？

```bash
# 查看错误信息
./dev.sh logs django

# 手动重启 Django
./dev.sh restart django

# 检查服务状态（更多命令）
./dev.sh help
```

## 脚本组织结构

新的脚本采用模块化组织，易于维护和扩展：

```
.
├── dev.sh                          ← 主入口（你用这个）
└── scripts/
    ├── dev.sh                      ← 命令路由器
    ├── dev/
    │   ├── start.sh                ← 启动服务
    │   ├── stop.sh                 ← 停止服务
    │   ├── restart.sh              ← 重启服务
    │   ├── logs.sh                 ← 查看日志
    │   └── clean.sh                ← 清理数据
    ├── db/
    │   ├── init.sh                 ← 数据库初始化
    │   └── query.sh                ← 数据库查询工具
    └── utils/
        ├── colors.sh               ← 彩色输出
        └── diagnostic.sh           ← 诊断工具
```

## 📊 常见数据库操作

### 使用内置工具连接数据库

```bash
# 列出所有数据库
source scripts/db/query.sh
psql_list_databases

# 连接到 PostgreSQL 交互式终端
psql_connect

# 执行 SQL 查询
psql_query postgres "SELECT * FROM auth_user LIMIT 5;"

# 备份数据库
psql_backup postgres ./my_backup.sql
```

### Django 迁移操作

```bash
# 创建新的迁移文件（修改模型后）
docker exec lesser-django python manage.py makemigrations

# 应用迁移到数据库
docker exec lesser-django python manage.py migrate

# 进入 Django 交互式 shell
docker exec -it lesser-django python manage.py shell
```

## ✅ 常见问题

**Q: 启动很慢？**
A: 第一次需要下载镜像，约 2-3 分钟。后续快速启动。

**Q: 代码改了但没有重载？**
A: Django 有自动热重载。如果未生效，运行 `./dev.sh restart django`

**Q: 想看实时日志？**
A: 运行 `./dev.sh logs` 查看所有服务日志

**Q: 如何完全重置？**
A: ⚠️ 会删除所有数据：
```bash
./dev.sh clean
./dev.sh  # 重新启动
```
