## 一、当前问题诊断

### 1. Frontend（Flutter）问题

#### 1.1 工程结构混乱
- **问题**：`lib/` 下的分层不清晰
  - `main.dart`, `app/`, `core/`, `features/`, `shared/` 混合，缺乏明确的分层
  - 组件划分不够细腻，导致某些文件/目录庞大
  - `theme` 管理分散，可能出现不统一的样式定义

- **现状**：
  ```
  lib/
  ├── main.dart              # 入口，混合了路由和初始化逻辑
  ├── app/                   # 不知道是应用配置还是路由
  ├── core/                  # 核心工具？业务模型？
  ├── features/              # UI+State混合
  └── shared/                # 共享组件+工具混合
  ```

#### 1.2 UI 组件库问题
- **问题**：没有明确的组件分级系统
  - 原子组件(Button, Input) 和页面级组件混放
  - 组件复用性差，相似功能重复实现
  - Theme/Style 定义分散

### 2. Django 后端问题

#### 2.1 缺乏文档和注释
- **问题**：没有清晰的说明文档
  - Django Project 和 Apps 的关系不明确
  - `settings.py`, `urls.py`, `wsgi.py` 的作用说明缺失
  - 没有说明为什么这样设计，新手难上手

#### 2.2 App 内部结构平面化
- **现状**：以 `users`, `chat`, `content`, `friend`, `setting_page` 为大 app
- **问题**：
  ```
  users/
  ├── models.py          # 所有模型混在一个文件
  ├── views.py           # 所有视图混在一个文件
  ├── urls.py
  ├── serializers.py     # 如果有的话
  ├── admin.py
  └── migrations/
  ```
  - 无法标识不同的功能模块
  - 模型、视图、序列化器没有分类
  - 难以维护和扩展

#### 2.3 Root Project 配置混乱
- `lesser_root_folder/` 名字不清晰
- Settings 中的配置没有分组注释
- 没有说明每个配置的作用

### 3. 基础设施问题（infra & scripts）

#### 3.1 脚本冗余且缺乏说明
- **现状脚本**：
  - `start-all.sh`, `start-dev.sh` 等放在根目录（冗余）
  - `/scripts/` 下也有 `start-local-*.sh`, `start-prod.sh`（重复）
  - 没有单一入口点，新人不知道应该运行哪个

- **问题**：
  - 无法单独启动数据库、单独启动 Django、联合启动等灵活组合
  - 脚本中缺少使用说明注释
  - 清理测试数据没有对应脚本

#### 3.2 Docker Compose 文件无说明
- **现状**：
  - `docker-compose.yml` （开发环境）
  - `docker-compose-apisix.yml` （API网关）
  - `docker-compose.prod.yml` （生产环境）
  - 没有一句话说明每个文件的用途

- **问题**：新人不知道应该使用哪个 Compose 文件

#### 3.3 网关架构混乱
- **现状**：同时存在 Traefik 和 APISIX 配置
  - `infra/traefik/` 和 `infra/apisix/` 并存
  - Docker Compose 中同时配置 Traefik
  - 不清楚迁移状态

- **问题**：
  - 配置重复，容易出现不一致
  - 开发时的路由规则不明确
  - 缺少 APISIX→etcd 的完整配置

### 4. 开发流程缺陷

#### 4.1 没有热重载
- **问题**：修改 Django 代码后需要手动重启服务
- **影响**：开发效率低，测试周期长

#### 4.2 缺乏新功能添加指南
- **问题**：新开发者不知道：
  - 在哪里定义数据模型（Django: `models.py` 还是按功能分文件？）
  - 在哪里定义 API 端点（Root URLs 还是 App URLs？）
  - 在哪里定义 Flutter UI（在 `features/` 哪个目录？）
  - 如何关联他们（API 和 UI 如何对应？）

#### 4.3 数据初始化混乱
- 没有清理脚本重置为初始状态
- 没有种子数据脚本
- 测试数据和真实数据混合

## 二、优化方案

### A. Frontend 分层改进

#### 新的目录结构
```
lib/
├── domain/              # 纯业务模型（无 Flutter/JSON 依赖）
│   ├── entities/        # 核心业务对象 (User, Post, Chat)
│   ├── repositories/    # 业务逻辑接口（由 data 层实现）
│   └── usecases/        # 用例/业务规则（可选）
│
├── data/                # 数据层（API/Repository 实现）
│   ├── datasources/     # API 调用、本地存储
│   ├── models/          # DTO/JSON 模型
│   ├── repositories/    # Repository 实现（继承 domain）
│   └── mappers/         # DTO ↔ Entity 映射
│
├── feature/             # 功能/UI 层（按功能模块组织）
│   ├── home/
│   │   ├── presentation/
│   │   │   ├── pages/
│   │   │   ├── widgets/
│   │   │   └── providers/ # Riverpod 状态管理
│   │   └── ...
│   ├── profile/
│   ├── chat/
│   ├── settings/
│   └── ...
│
├── app/                 # 应用级配置
│   ├── routing/         # 路由定义
│   ├── theme/           # 统一 Theme 管理
│   ├── di/              # 依赖注入（Riverpod providers）
│   └── config/          # 应用配置常量
│
├── shared/              # 跨功能共享资源
│   ├── widgets/         # 通用 UI 组件（Button, Input, Card 等）
│   ├── utils/           # 工具函数
│   ├── constants/       # 全局常量
│   └── extensions/      # 扩展方法
│
└── main.dart
```

#### 改进点
1. **明确分层责任**：
   - `domain` 层：纯 Dart，不依赖 Flutter，便于单元测试
   - `data` 层：数据获取和映射，依赖 `domain`
   - `feature` 层：UI 和状态管理，依赖 `domain` 和 `data`
   - `app` 层：路由、主题、依赖注入

2. **统一 Theme 管理**：
   - `app/theme/` 集中定义所有主题相关
   - 颜色、排版、圆角等在一处管理
   - 支持深色/浅色主题切换

3. **细粒度组件划分**：
   - 原子组件（基础 UI）→ `shared/widgets/base/`
   - 业务组件 → 各 `feature/*/presentation/widgets/`
   - 复合组件 → 单独目录和文件

**改进说明**（添加注释）：
```python
# models/user.py
"""
用户模型定义

Django ORM 模型定义了数据库表结构和字段验证。
这里包含：
- User: 用户身份和认证
- UserProfile: 用户个人信息（可选，用于扩展 Django User）
"""

from django.db import models
from django.contrib.auth.models import AbstractUser

class User(AbstractUser):
    """
    用户模型 - 继承 Django 内置 User
    
    字段:
        - username: 用户名（唯一）
        - email: 邮箱（认证用）
        - first_name, last_name: 名字
        - created_at: 创建时间
    """
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    class Meta:
        db_table = 'users_user'
        verbose_name = '用户'
        verbose_name_plural = '用户'

class UserProfile(models.Model):
    """
    用户资料 - 扩展用户基本信息
    
    为了不修改 User 表，采用 1:1 关系扩展
    """
    user = models.OneToOneField(User, on_delete=models.CASCADE, related_name='profile')
    bio = models.TextField(blank=True)
    avatar = models.ImageField(upload_to='avatars/', null=True)
```

示例：添加用户获取端点

```python
# models/user.py
class User(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField()

# serializers/user.py
class UserSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ['id', 'name', 'email']

# views/user.py
from rest_framework.viewsets import ModelViewSet

class UserViewSet(ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSerializer

# urls.py
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import UserViewSet

router = DefaultRouter()
router.register('', UserViewSet, basename='user')
urlpatterns = [path('', include(router.urls))]
```

  final String name;
  final String email;
  User({required this.id, required this.name, required this.email});
}

// data/models/user_model.dart
class UserModel {
  final int id;
  final String name;
  final String email;
  factory UserModel.fromJson(Map json) => UserModel(
    id: json['id'],
    name: json['name'],
    email: json['email'],
  );
}

// data/repositories/user_repository.dart
class UserRepository {
  Future<List<User>> getUsers() async {
    final response = await http.get('/api/users/');
    return (response.data as List).map((u) => User(...)).toList();
  }
}

// feature/user/presentation/providers/user_provider.dart
final userListProvider = FutureProvider((ref) {
  return ref.watch(userRepositoryProvider).getUsers();
});

## 热重载

修改 Django 代码后，服务会自动重启（`.sh` 脚本中使用 `--reload` 标志）。

修改 Flutter 代码后，使用 `r` 热重载，`R` 热重启。

## 常用命令

```bash
# 查看日志
docker-compose logs -f django      # Django 日志
docker-compose logs -f redis       # Redis 日志

# 运行 Django 管理命令
docker-compose exec django python manage.py <command>

# 创建数据库迁移
docker-compose exec django python manage.py makemigrations

# 应用迁移
docker-compose exec django python manage.py migrate

# 创建超级用户
docker-compose exec django python manage.py createsuperuser
```

## 常见问题

**Q: 启动时报数据库连接错误？**
A: 确保 PostgreSQL 容器已启动，可用 `docker-compose ps` 检查。

**Q: Flutter 编译失败？**
A: 运行 `flutter clean` 后重试，或检查 Flutter SDK 版本。

**Q: API 请求 404？**
